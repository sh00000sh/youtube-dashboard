# 유튜브 인게이지먼트 대시보드 — 개발 로그

> 유진투자선물 유튜브 채널의 성과를 자동 수집·시각화하는 운영 대시보드.
> 유튜브 데이터를 자동으로 가져와 구글 시트에 기록하고, KPI 대비 성과를 한눈에 보여준다.

**라이브:** https://yujinytdb.up.railway.app

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| **목적** | 채널 운영 지표(조회수·구독자·참여율·KPI 달성률)를 자동으로 모아 보고, 콘텐츠 의사결정에 활용 |
| **사용자** | 채널 PD/운영자 1인 |
| **핵심 가치** | 스튜디오를 매번 들어가 손으로 적던 KPI 트래킹을 **자동 수집 + 실시간 시각화**로 전환 |

### 한 줄 요약
> "유튜브 스튜디오를 매번 열어 손으로 적던 지표를, 버튼/자동 스케줄로 구글 시트에 쌓고 토스 스타일 대시보드로 보는 시스템"

## 2. 기술 스택 & 아키텍처

**Stack**
- **백엔드**: Node.js + Express
- **프론트엔드**: Vanilla JS + Chart.js (빌드 과정 없는 단일 HTML)
- **데이터 저장**: Google Sheets (단일 진실 공급원)
- **외부 API**: YouTube Data API v3, YouTube Analytics API, Google Sheets API
- **인증**: OAuth 2.0(채널 분석/비공개 영상) + 서비스 계정(시트 쓰기)
- **호스팅/배포**: Railway (GitHub 연동 자동 배포) + PostgreSQL

**데이터 흐름**
```
[YouTube]
  ├─ 공개 지표(조회/좋아요/댓글/구독자) ── YouTube Data API (API Key)
  └─ 심층 지표(평균조회율·시청시간·구독전환·트래픽소스) ── YouTube Analytics API (OAuth)
        ↓ 수집
[Node/Express 서버 (Railway)]
        ↓ 기록
[Google Sheets] ── 단일 데이터 소스 (일별/영상별/스냅샷 탭)
        ↓ 읽기
[대시보드 (브라우저)] ── Chart.js 시각화
```

**설계 의사결정 (배운 점)**
- 프론트+백엔드를 한 앱에 합쳐 CORS·복잡한 빌드 제거 → 초기 배포 난이도 ↓
- DB 대신 **구글 시트를 데이터 소스로** 선택 → 비개발자도 직접 데이터를 보고 수정 가능
- 시크릿(서비스계정 JSON)은 **Base64 인코딩** 후 환경변수로 주입 → 줄바꿈/따옴표 깨짐 문제 해결

## 3. 주요 기능

### 데이터 수집
- **자동 동기화**: 채널 일별 지표 + 영상별 지표를 유튜브에서 가져와 시트에 기록
- **영상 자동 수집**: 채널 업로드 목록에서 영상을 자동으로 끌어옴(수동 ID 입력 불필요)
- **시간별 스냅샷 로봇**: 업로드 후 24시간 동안 1시간 간격으로 조회수 기록(서버 내 스케줄러)
- **업로드 시각 저장**: 시간대별 성과 분석을 위해 게시 시각을 별도 기록

### 시각화 & 분석
- **KPI 달성률**: 일별/영상별/누적, 기간 선택(7·28·90일·전체), 색상(달성/미달/완전미달)
- **누적 성장 추이**: 지표별 누적 곡선 + KPI 목표선 + 현재 페이스 예측선
- **CTR × 평균조회율 산점도**: "낚시성 영상" 감별(클릭은 되나 이탈하는 영상)
- **숏폼/롱폼 분리 분석** 및 포맷별 성과 비교
- **과거 vs 현재 비교**: 트래픽 소스로 **광고 유입 자동 제외**, 순수 유기적 성과만 비교 + 포맷별 신규 KPI 기준점 도출
- **요일별/시간대별 업로드 성과**: 언제 올리는 게 효과적인지
- **영상 상세 시계열**: 제목/썸네일 클릭 시 우측 재생 + 좌측 게시 후 추이(24h·7·28일·전체)

### UX
- **토스 스타일 디자인**: 절제된 색감(블루 단색), 보더리스 카드+그림자, 다크모드
- **섹션 토글**로 긴 페이지를 그룹별로 압축(KPI·일별·영상 그룹)
- **스크롤 고정 요약바**, **D+운영일수 배지**, 참여율 컬럼·썸네일

## 4. 개발 타임라인 (주요 마일스톤)

| 단계 | 내용 |
|---|---|
| **1. MVP** | 유튜브 조회수/좋아요/댓글을 가져와 표로 보여주는 최소 버전 (Railway + Postgres) |
| **2. 채널 자동 수집** | 영상 ID 수동 입력 → 채널 업로드 목록 자동 수집으로 전환 |
| **3. 구글 시트 전환** | 데이터 저장소를 DB → 구글 시트로. OAuth + 서비스계정 인증 구축 |
| **4. 심층 지표 연동** | YouTube Analytics API(OAuth)로 평균조회율·시청시간·구독전환 수집 |
| **5. 시각화 고도화** | KPI 색상·차트·누적/예측·산점도·요일/시간대 분석 추가 |
| **6. 시간별 스냅샷** | 업로드 직후 24시간 1시간 간격 자동 기록 스케줄러 |
| **7. 토스 디자인 적용** | 색감·투명도·카피·다크모드 전면 리디자인 |
| **8. 섹션 토글 구조** | 긴 페이지를 3개 그룹 토글로 압축 |
| **9. 과거 vs 현재 비교** | 트래픽 소스로 광고 유입 자동 제외, 유기적 성과 비교 + KPI 기준점 도출 |
| **10. 인프라 이전** | 배포/저장소를 개인 계정으로 이전(Fork + Railway 재구축) |

## 5. 기술적으로 해결한 문제 (Troubleshooting)

| 문제 | 원인 | 해결 |
|---|---|---|
| 서비스계정 JSON 환경변수 깨짐 | 줄바꿈·따옴표가 env에서 손상 | **Base64 인코딩**으로 단일 문자열화 후 디코딩 |
| OAuth `invalid_grant` | Playground에서 "내 자격증명 사용" 미체크 | 본인 클라이언트로 토큰 재발급 |
| `[채널분석] Forbidden` | 채널 소유 계정이 아닌 계정으로 인증 | 채널 소유 계정으로 재인증 + 앱 게시(테스트→프로덕션) |
| 비공개 과거 영상 정보 접근 불가 | yt-analytics 스코프만으론 비공개 영상 메타 못 읽음 | OAuth에 `youtube.readonly` 스코프 추가 |
| 최근 2~3일 데이터 없음 | YouTube Analytics 데이터 확정 지연 | 정상 동작임을 명시(지연 배지 표시) |
| 노출수·CTR 자동 수집 불가 | 유튜브가 API로 미제공(스튜디오 전용) | 한계 명시 + 수동 입력 칸 제공 |
| 과거 비교 쿼리 미지원 | `video` 차원 + 트래픽소스 필터 동시 사용 불가 | 영상별 개별 조회 + **병렬 처리(10개씩)**로 속도 개선 |
| 다크모드 그래프 눈부심 | 차트 격자선 색이 고정(밝은 회색) | 테마 감지 후 격자/목표선 색 동적 전환 |

## 6. 데이터로 도출한 인사이트 (실제 활용)

- **광고 제외 유기적 비교** 결과, 현재 숏폼 조회수가 과거 평균을 상회 → 유입은 정상
- **숏폼 vs 롱폼 구독전환 차이**(숏폼 2.3 vs 롱폼 6.3, 약 3배) 발견 → 롱폼을 구독자 확보 채널로 포지셔닝
- 과거 숏폼 실측치(조회수 989·조회율 70.7%·구독전환 2.31)를 **신규 KPI 기준점**으로 역산해 대시보드 KPI 전면 재설정

## 7. 포트폴리오 어필 포인트

- **0 → 1 풀스택 구축**: 기획·API 연동·인증·배포·시각화까지 단독 수행
- **실데이터 기반 의사결정**: 단순 대시보드가 아니라 KPI 도출·콘텐츠 전략으로 연결
- **외부 API 한계를 다루는 능력**: 데이터 지연·미제공 지표·미지원 쿼리를 우회/명시
- **비개발자 친화 설계**: 구글 시트를 소스로 삼아 운영자가 직접 데이터를 다룰 수 있게
- **운영 이관 경험**: 인증 토큰/환경변수 손실 없이 배포 인프라를 개인 계정으로 이전
